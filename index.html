협업 기능 로드맵
현재 ProjectOS는 개인 브라우저 저장소에 데이터를 저장합니다. 협업형 제품으로 확장하려면 로그인, 워크스페이스, 권한, 실시간 동기화, 대화 기록을 백엔드에 올려야 합니다.
1
Kakao Login + 사용자 프로필
카카오 OAuth로 가입/로그인하고 서비스 내부 userId와 연결합니다. 닉네임, 프로필 이미지는 동의 항목 기준으로 저장합니다.
MVP
2
Workspace 초대와 역할
Owner, Manager, Member, Viewer 권한을 두고 프로젝트별 초대 링크와 만료 시간을 관리합니다.
Backend
3
문맥형 댓글과 멘션
Requirement, Task, Test Case, API Spec에 댓글 스레드를 붙이고 @멘션으로 담당자에게 알림을 보냅니다.
Realtime
4
팀 대화와 의사결정 로그
일반 채팅보다 중요한 것은 결정사항 추적입니다. 대화에서 결정, 액션 아이템, 리스크를 추출해 기존 모듈과 연결합니다.
Core
5
KakaoTalk 초대와 알림
카카오톡 메시지 API로 초대/멘션/마감 알림을 보낼 수 있지만 친구 목록과 메시지 전송 권한 심사가 필요합니다.
Review
Kakao Integration
카카오로 묶을 수 있는 기능과 제한
OAuth + Talk
K
카카오 로그인
REST API 또는 JavaScript SDK로 구현 가능합니다. 실제 서비스는 토큰 교환을 서버에서 처리하는 편이 안전합니다.
가능
F
친구 선택 초대
친구 피커/친구 목록은 앱 권한이 필요하고, 기본적으로 같은 서비스에 연결된 사용자 중심으로 제한됩니다.
심사
M
카카오톡 메시지
초대, 멘션, 마감 알림을 보낼 수 있습니다. 메시지 API 권한과 사용자 동의가 필요합니다.
심사
C
톡캘린더 일정
스프린트 리뷰, 마감일, 회의 일정 연동 후보입니다. 권한 범위와 테스트 제한을 별도로 확인해야 합니다.
